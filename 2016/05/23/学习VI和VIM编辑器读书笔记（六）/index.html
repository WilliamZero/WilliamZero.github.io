自定义vi

使用ex命令：set 在vi中改变选项。
vi被打开，就会读入主目录中的.exrc文件。也可以读入当前目录的.exrc文件（会覆盖主目录的重复选项）

若文件不存在，可自己手动创建。

：set命令

：set命令可以更改两种类型的选项：一种是切换选项，只能选择是或否；另一种可接受数值或者字符串。
：set all 显示选项的完整列表。
对于切换选项
：set option 选项打开
：set nooption 选项关闭
：set option？ 查找选项的当前值

单词缩写

定义缩写，使用ex命令：ab abbr phrase

使用map命令

：map x sequence 定义字符x映射到一个编辑命令sequence
：unmap x 取消x的映射
：map 列出所有被映射的字符
：help map map帮助信息

个人感觉不必深究，明白.vimrc中的一些配置即可。之后会解析一个目前在用的一个配置文件。

保护按键免被ex解释

ENTER、ESC、BACKSPACE、DELETE 这些按键已经在ex中有定义了。如果想将这类按键作为命令序列的一部分，必须在前面加上CTRL-V，转换按键的正常意义。

要将控制字符作为映射命令，在大部分情况下，只需要按着CTRL键并同时按下字母键。但有三个控制字符必须用^（CTRL-V）做转义，即为^T,^W,^X。
：map CTRL-VCTRL-T sequence

复杂的映射范例

定义复杂映射最好的方法是先手动编辑，写下每个按键组合；接着用按键组合创建映射键。

更多映射按键的范例

1.当移动到单词结尾时加入文本：
：map e ea
2.对调两个单词
：map K dwwP
3.保存一个文件并编辑下一个文件
：map q ：w^M:n^M
4.重复地寻找与删除一个单词或词组周围的括号
：map = xf）xn
前提：先用/（ 加回车找到了一个左括号。

插入模式中的映射键

在map命令后加上感叹号（！），即可强制覆盖按键原来的意义，以产生插入模式中的映射行为。
eg： ：map！ % ^[kA:^[jA
在当前任意位置输入%，会使上一行结尾加上：
解除插入模式的映射键
：unmap！ 字符

使用标签（tag）

ctags命令在Unix命令行上执行。它的功能是产生一个信息文件，让vi可用该文件判断各个文件中分别定义了哪些函数。
默认情况下，这个文件名为tags。在vi中，下面的命令
：!ctags file.c
会在当前目录中创建一个名为tags的文件，其中包含file.c 中定义的所有函数的信息。
：!ctags *.c
会创建描述所有C源代码文件中的tags文件。
如果想观察或编辑程序中的某个函数，则在vi中，使用
:tag name
会在tags文件中寻找哪一个文件包含了函数name的定义。接着vi读入这个文件，并将光标指到定义name函数的那一行。

在vi的命令模式中使用标签功能
将光标移到要查找的标识符（开头）上，再输入 ^] 。vi会执行标签查找，并将光标移到定义了标识符的文件中。（先将当前编辑文件保存）
