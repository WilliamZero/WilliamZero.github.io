越过基础的藩篱

更多命令组合

更改 删除 复制 从光标位置到……
cH dH yH 屏幕顶端
cL dL yL 屏幕底端
c+ d+ y+ 下一行
c5| d5| y5| 本行的第5列
2c） 2d） 2y） 往下第二个句子
c{ d{ y{ 上一段
c/pattern d/pattern y/pattern pattern（模式）
cn dn yn 下一个模式
cG dG yG 文件结尾
c13G d13G y13G 第13行

打开vi的选项

前进到特定位置

vi +n file 在第n行打开file
vi + file 在最后一行打开file
vi +/pattern file 在第一次出现pattern的地方打开file

如果模式中包含空格，则需要单引号或双引号括起整个模式，或者用反斜线将空格转义。
+/pattern 在你编辑到一半却必须离开时很有用。你可以将当前位置用某个模式标记起来，如ZZZ或HEAR。当你回来时，只要记得/ZZZ或/HEAR就可以了。

####只读模式

vi -R file
或 view file
若想要更改文件内容，可以在write命令（：w）后面加上感叹号，以覆盖掉只读模式：
：w！

####恢复缓冲区
恢复系统死机时的编辑缓冲区
vi -r file
（推荐使用vi的autowrite与autosave选项）

###善加利用缓冲区
编辑时的最后一次删除或拖拽的内容会保存到缓冲区中。我们可以访问这些缓冲区并使用放置命令（p或P）将这些保存的文本放回文件中。

vi会将最后9次的删除操作保存在编号的缓冲区中，你可以访问其中任何一个，以恢复任何一次（或所有）的删除操作。

vi也可以让你将拖拽（复制）的文本放在依字母标识的缓冲区中。拖拽的文本可以被保存到26个缓冲区（a~z）中，并且可在编辑会话的任何时候，使用放置命令来恢复这些文本。

####恢复删除

最后一次删除的内容放在缓冲区1，倒数第二次的则存放在缓冲区2。。。

“np 将缓冲区n（1~9）的内容恢复到光标后。

“1pu.u.u 以此类推 逐一将每个缓冲区的内容放置到文件中。每一次输入u时，恢复的文本会被移除。输入点号时，则把下一个缓冲区的内容恢复到文件中。不断输入u与.，直到找到所需的文本为止。

####将文本拖拽到命名缓冲区
“dyy 将当前行复制到缓冲区d中

“dp 将缓冲区d的内容放置在光标后
“dP 将缓冲区d的内容放置在光标前

（使用命名缓冲区在文件间传递部分文本）

使用大写字母指定缓冲区名称，则复制或删除的文本会追加到当前缓冲区中。
“Z3yy 将3行的内容追加到缓冲区z中

####在一处做标记

在一个vi会话中，你可以在某处做一个看不见的”书签”，然后在别处编辑，完成后再回到书签位置。可在命令模式中输入：
mx 将当前位置标记成x（x可以是任何字符）
‘x （单引号）将光标移到标记x所在行的第一个字符
`x （反引号）将光标移到以x标记的字符
`` （两个反引号）在移动位置后，回到上一个标记或上下文的确切位置
‘’ （两个单引号）回到上一个标记或上下文所在行的开头
注意：标记只有在当前vi会话中有用。
